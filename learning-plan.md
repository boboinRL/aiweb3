# Personal Learning Plan — AI × Web3 School

## Goal

Build practical AI × Web3 capability for quant/trading workflows: understand AI agents, Web3 data/transactions, wallets/permissions, on-chain context, and risk controls well enough to design small proof-of-work experiments.

## Constraints

- Daily time budget: 2 hours
- Output language: English
- Repository style: public proof-of-work, no secrets or sensitive trading credentials

## 4-Week Roadmap

### Week 1 — Shared Language and Trading Map

Focus: AI basics + Web3 primitives + market context.

- AI: LLM, Prompt, Context, RAG, Agent
- Web3: Network, Wallet, Smart Contract, DeFi, Indexing, Security
- Output: a market map of where AI can help Web3 trading decisions

### Week 2 — Data and Tooling

Focus: turn public data into usable research context.

- Study: Chain-aware Context, Web3 Tool Use, Indexing, Oracle
- Experiment: collect or manually inspect one on-chain dataset / DEX pair / protocol metric
- Output: one notebook/script or structured research note

### Week 3 — Agent Workflow and Risk Controls

Focus: design agents that assist but do not blindly execute.

- Study: Agent Workflow, Agent Wallet, Account Abstraction, AI Security
- Experiment: draft a human-in-the-loop trading assistant workflow
- Output: risk checklist and execution boundary document

### Week 4 — Proof-of-Work Project

Focus: small demo suitable for portfolio or hackathon continuation.

Candidate directions:

1. AI-assisted on-chain research digest
2. DEX pair monitoring and thesis generator
3. Wallet/risk permission policy explainer
4. DeFi protocol risk signal dashboard
5. Trading journal agent with post-trade review prompts

## Daily 2-Hour Structure

### Minimum Path — 60 minutes

- 20 min: read one Handbook section
- 20 min: summarize key concepts in English
- 20 min: write one actionable question or feedback item

### Recommended Path — 120 minutes

- 30 min: read Handbook + WCB task page
- 40 min: do one small experiment or market/protocol research task
- 30 min: write daily note and check-in draft
- 20 min: extract Handbook feedback or next action

### Challenge Path — 150+ minutes

- Add a reproducible artifact: script, notebook, chart, dataset link, prompt trace, or transaction/risk analysis.

## Weekly Review Questions

- What did I learn that improves trading judgment?
- What can be automated safely, and what must remain human-approved?
- What data source, model output, or protocol assumption could be wrong?
- What proof-of-work artifact did I create this week?
